# Fix SCSS Style: Remove BEM, Use Nesting

**Date:** 2026-05-04

## Goal

Remove all BEM naming conventions from the SCSS and HTML, replacing them with SCSS nesting and short contextual class names. Align the card component with the convention already used by `.button` and `.contact`.

## Scope

Only `_card.scss`, `_card.erb`, and `_card_grid.erb` contain BEM. All other components (`.button`, `.contact`, `header`, `footer`, `nav`) already follow the nesting convention.

## Convention

- **Sub-elements** get short, single-word class names scoped by SCSS nesting. The CSS selector is generated by the nesting; the HTML class is just the short name.
- **Modifiers** become compound classes on the root element (e.g. `class="card pink"`), matching the existing `.button.pink` pattern.
- **Semantic elements** (`<header>`, `<footer>`) are preferred over classed divs where appropriate.

## Changes

### `styles/_card.scss`

- Move the `@each` color loop inside `.card {}`.
- Change `.card--#{$name}` to `&.#{$name}` (generates `.card.pink`, `.card.grey`, etc.).
- Change `a.card--#{$name}:visited` and `a.card--#{$name}:hover` to `a&.#{$name}:visited` / `a&.#{$name}:hover`, nested inside `.card {}` (generates `a.card.pink:visited` etc., preserving link-only hover behaviour).
- Inside the loop, change `.card__icon` and `.card__title` references to `.icon` and `.title`.
- Move `.card__header`, `.card__icon`, `.card__title`, `.card__content`, `.card__access` inside `.card {}` as `.header`, `.icon`, `.title`, `.content`, `.access`.
- Change `.card--wide-title` to `&.wide-title` nested inside `.card {}`.
- Replace the standalone `.card-grid-header` / `.card-grid-footer` rules with `header` and `footer` element selectors nested inside `main {}`. Styling lives in `main.scss`.

### `source/partial/_card.erb`

- Change class-building logic:
  - `"card card--#{card_colour}"` → `"card #{card_colour}"`
  - `" card--wide-title"` → `" wide-title"`
- Change element class attributes:
  - `card__header` → `header`
  - `card__icon` → `icon`
  - `card__title` → `title`
  - `card__content` → `content`
  - `card__access` → `access`

### `source/partial/_card_grid.erb`

- Replace `<div class="card-grid__header">` with `<header>`.
- Replace `<div class="card-grid__footer">` with `<footer>`.

No wrapper is needed. The card-grid partial renders inside `<main>` (via `yield` in `layout.erb`), making `<header>` and `<footer>` children of `<main>`. The site's own `<header>` is a sibling of `<main>`, so `main { header { ... } }` in SCSS scopes correctly without conflict.

## No changes required

- Page-level ERB files — `colour: 'pink'` locals are passed unchanged; the partial consumes them differently.
- `_button.scss` — already uses `&.#{$name}` nesting with no BEM.
- `_contact.scss` — already uses short contextual names (`.channel`, `.label`, `.note`).
- `_header.scss` — already uses nesting with non-BEM class names.
- `main.scss` — add `header { margin-bottom: 1rem; }` and `footer { margin-top: 1rem; }` nested inside the existing `main {}` block.

After changes, run `./manage-sgi.sh --build app` and confirm the build succeeds with no SCSS errors. Extract and visually verify card colours and layout are unchanged.
